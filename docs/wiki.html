1. Genel Bakış

turkce-sozluk-lab, kullanıcıların kelime tanımları, örnek cümleler, eş anlamlılar, zıt anlamlılar, deyimler, alıntılar ve benzeri sözlük öğelerine ulaşmasını sağlayan Node.js + MySQL tabanlı bir web uygulamasıdır. Projede:

  • Kullanıcı Arayüzü (Public): Bir metin kutusundan (AJAX veya fetch kullanarak) kelime araması yapılabilir. Arama sonucu, kelimenin anlam(lar)ı, örnek cümle(ler)i, köken bilgisi, eş/zıt anlam vb. ilişkili içerikler görüntülenir.
  • Admin Paneli: Kelime ekleme, güncelleme, definition (anlam) yönetimi, kelimeler arası ilişkiler (synonym, antonym, idiom, collocation, quote) tanımlama gibi işlemleri yapabileceğiniz, sadece yetkili kullanıcıların erişimine açık bir panel mevcuttur.
  • Veritabanı (MySQL): words, definitions, relations gibi tablolar üzerinden kelimelerle ilgili tüm verileri saklar ve yönetir.
  • Full-text Search: Belirli sütunlarda tam metin araması yaparak kullanıcıların kelime veya tanım aramalarını kolaylaştırır. Ayrıca LIKE tabanlı bir arama rotası da mevcuttur.

Proje, “Türkçe Sözlük Uygulamaları” alanında eğitim, araştırma veya hobi amaçlı geliştirilebilir, genişletilebilir bir temel sunar.

2. Kurulum ve Konfigürasyon

2.1 Gerekli Bağımlılıklar

  • Node.js (v14 veya üstü)
  • MySQL (5.7 veya üstü)
  • İsteğe bağlı: Git, Docker vb. (Dağıtım opsiyonlarına bağlı)

2.2 Projenin Çalıştırılması

1. Kaynak Kodun Temini

git clone https://github.com/kullanici/turkce-sozluk-lab.git
cd turkce-sozluk-lab

2. Bağımlılıkların Yüklenmesi

npm install

3. MySQL Veritabanı Kurulumu

- sozluk_db adında bir veritabanı oluşturun.

CREATE TABLE words (
  id INT AUTO_INCREMENT PRIMARY KEY,
  word VARCHAR(255),
  origin TEXT,
  frequency_tags VARCHAR(255),
  audio_url VARCHAR(255)
);

CREATE TABLE definitions (
  id INT AUTO_INCREMENT PRIMARY KEY,
  word_id INT,
  definition_text TEXT,
  example_sentence TEXT,
  part_of_speech VARCHAR(50),
  sense_label VARCHAR(50),
  FOREIGN KEY (word_id) REFERENCES words(id)
);

CREATE TABLE relations (
  id INT AUTO_INCREMENT PRIMARY KEY,
  left_word_id INT,
  right_word_id INT,
  relation_type VARCHAR(50),
  FOREIGN KEY (left_word_id) REFERENCES words(id),
  FOREIGN KEY (right_word_id) REFERENCES words(id)
);

- Tam metin araması için ek indeksler de ekleyebilirsiniz (örn. ALTER TABLE words ADD FULLTEXT(word);).

4. .env Dosyasının Düzenlenmesi

DB_HOST=127.0.0.1
DB_USER=root
DB_PASSWORD=mysql_sifreniz
DB_NAME=sozluk_db
ADMIN_PASSWORD=gucluSifre
SESSION_SECRET=oturumAnahtari
PORT=3000

İhtiyaç halinde DB_HOST, DB_USER, DB_PASSWORD, DB_NAME gibi değerleri değiştirin.

5. Sunucuyu Başlatma

npm start
# veya
node index.js

Varsayılan olarak PORT=3000 ayarlıysa, tarayıcıda http://localhost:3000 üzerinden uygulamaya erişebilirsiniz.

3. Kullanım Rehberi

3.1 Genel Kullanıcı Arayüzü

• Anasayfa (Public): public/index.html dosyası, kelime araması yapılan temel arayüzdür. Kullanıcı, arama kutusuna kelime girerek GET /search-like veya GET /search endpoint’i üzerinden sorgulama yapabilir. Sonuçta gelen kelime bilgileri, tanımlar, örnek cümleler vb. ekranda listelenir.

• Dinamik İçerik: Sinonimler, antonimler, deyimsel kalıplar vb. /api/synonyms, /api/antonyms, /api/idioms gibi uç noktalardan çekilip sayfada gösterilir.

3.2 Admin Paneli

Admin paneli, public/admin.html (veya benzer) bir HTML dosyası üzerinden kullanılabilir. Giriş yapmak için:

  1. Tarayıcıda http://localhost:3000/admin/login sayfasına gidin.
  2. Kullanıcı adı: admin, Şifre: .env dosyasında ADMIN_PASSWORD değeri.

Başarılı giriş sonrası /admin rotasına yönlendirilebilir ya da admin.html gibi bir sayfa üzerinden panele erişebilirsiniz.

Paneldeki Başlıca Özellikler:

  • Kelime Yönetimi: “Kelimeler” sekmesinde yeni kelime ekleme, mevcut kelimeleri listeleme, sayfalama/sıralama, kelime düzenleme (origin, frequency_tags, audio_url) ve kelime silme işlemleri (silme route’u ileride eklenecek notu var ise şimdilik devre dışı olabilir).
  • Definition Yönetimi: “Definitions” sekmesinden bir kelimeye anlam (definition) ekleme, örnek cümle girme, var olan definition kayıtlarını listeleme, silme, düzenleme (PUT /admin/definitions/:id).
  • İlişkiler Yönetimi: “Relations” sekmesi; sinonim, antonim, deyim, alıntı vb. tüm relation_type çeşitleri eklenebilir, listelenebilir veya silinebilir. Bu sayede bir kelimenin diğeriyle olan bağı tanımlanır.
  • Alfabetik Liste: Harf seçerek o harfle başlayan kelimeleri listeleyen bir sekme. GET /words-by-letter?letter=a gibi uç noktayı çağırır.
  • Dashboard: Son eklenen kelimeler vb. özet istatistikler (kelime sayısı, definition sayısı vb.) gösterir.

Güvenlik: Admin paneline giriş yapmadan /admin/ ve türevi rotalara istek atılamaz. checkAdmin middleware’i session’daki isAdmin bayrağını kontrol eder, yoksa 403 hatası döner.

4. API Endpointleri

4.1 Kullanıcı Görebileceği Uç Noktalar

  1. GET /search - Full-text arama. Sorgu parametresi ?kelime=.... words ve definitions tablolarında MATCH...AGAINST ile arama yapar, sonuçları HTML olarak döndürür.
  2. GET /search-like - LIKE tabanlı arama. ?kelime=... şeklinde parametre alır. JSON formatında { results: [...] } döndürür.
  3. GET /api/synonyms?word=... - Belirtilen kelime için relation_type='synonym' kayıtlarını JSON olarak döndürür. Dönen veride synonyms array’i vardır.
  4. GET /api/antonyms?word=... - Aynı mantıkla zıt anlamlı ilişkileri listeler (relation_type='antonym').
  5. GET /api/idioms?word=... - Deyimsel kalıplar (relation_type='idiom').
  6. GET /api/quotes?word=... - Alıntılar (relation_type='quote').
  7. GET /api/collocations?word=... - Eşdizimler (relation_type='collocation').
  8. GET /api/word-family?word=... - Kelime ailesi (relation_type='word_family').
  9. GET /words-by-letter?letter=a - Belirli harfle başlayan kelimeleri listeler. Örneğin letter=a => apple, aradaki, vb. JSON formatı { words: [...] }.

4.2 Admin Panel Uç Noktaları

Not: Bu uç noktalar checkAdmin middleware’ine takılır. Yani, sadece admin oturumu (session) olan kullanıcılar erişebilir.

  • Giriş/Çıkış
    • GET /admin/login (form sayfası), POST /admin/login (giriş işlemi), GET /admin/logout (oturum sonlandırma).
  • Kelime Yönetimi
    • POST /admin/add-word: Kelime ekleme (word, origin, frequency_tags, audio_url).
    • GET /admin/words: Tüm kelimeleri sayfalama & sıralama ile listeler (?page=1&sort=id_desc).
    • GET /admin/words/:id: Tek bir kelimenin ayrıntısını döndürür (+ ilk definition).
    • PUT /admin/words/:id: Kelime günceller.
    • DELETE /admin/words/:id: (Henüz implement edilmediyse ileride eklenecek.)
  • Definitions
    • GET /admin/definitions: Tanımları (definitions) sayfalar halinde listeler. Kelime bilgilerini de JOIN ile getirir.
    • POST /admin/add-definition: Var olan bir kelimeye yeni definition ekler.
    • PUT /admin/definitions/:id: Definition günceller.
    • DELETE /admin/definitions/:id: Definition siler.
  • Relations
    • GET /admin/relations: Tüm relation kayıtlarını listeler (synonym, antonym, vb.).
    • POST /admin/relations: İki kelime adı ve relation_type alıp tabloya kayıt ekler.
    • DELETE /admin/relations/:id: Belirli bir ilişkiyi siler.
  • Dashboard
    • GET /admin/last-words: Son eklenen 10 kelime (JOIN definitions ile). Admin panelin “Son Eklenenler” kısmında kullanılır.

5. Dağıtım Rehberi

Proje, Node.js ve MySQL gerektirdiği için herhangi bir sunucu/hosting ortamında veya bulut platformda çalıştırılabilir. Ana mantık:

  1. MySQL Sunucusu: Mevcutsa DB_HOST, DB_USER, DB_PASSWORD, DB_NAME bilgilerini .env içine girin.
  2. Node.js Ortamı: Kodun çalışacağı makinede Node.js kurulu olmalıdır.
  3. .env Dosyası: Sunucuda .env ayarlarını doğru yapın (admin şifre, session_secret, port, vb.).
  4. Uygulamayı Başlatma: Gerekirse npm install ve npm run start komutlarıyla sunucuyu başlatın.

Docker ile Dağıtım (Öneri):

  • Bir Dockerfile yazarak (veya mevcudu düzenleyerek) Node.js ortamını container içine alabilir, .env ayarlarını da docker-compose.yml üzerinden yönetebilirsiniz.
  • docker-compose ile MySQL + Node.js servisini birlikte çalıştırabilir, böylece tek komutla projenizi ayağa kaldırabilirsiniz.
  • AWS ECS, Azure Container Instances gibi platformlara Docker imajınızı push edip kolayca ölçekleyebilirsiniz.

Heroku (Klasik) Dağıtım:

  • Heroku üzerinde MySQL yerine ClearDB eklentisi veya benzer bir MySQL hizmeti kullanabilirsiniz.
  • .env içeriğini, Heroku’nun “Config Vars” kısmına ekleyerek ayarlamanız gerekir.
  • Sonra git push heroku main veya “Deploy via GitHub” yöntemiyle uygulamayı Heroku’ya gönderirsiniz.

AWS:

  • Elastic Beanstalk: Node.js platformu seçip eb deploy komutlarıyla gönderebilirsiniz. MySQL için AWS RDS kullanarak DB_HOST, DB_USER, DB_PASSWORD gibi environment variable’ları tanımlamalısınız.
  • ECS / Fargate: Docker tabanlı dağıtım yaparak container orkestrasyonunu AWS yönetebilir.
  • EC2 Üzerinde Manuel: Bir EC2 instance’ına SSH ile girip Node.js & MySQL kurulumları yapıp projeyi manuel çalıştırabilirsiniz. Ancak otomasyon için Docker veya EB daha uygundur.

Ek Güvenlik: Canlı ortamda HTTPS zorunlu tutulmalı, admin paneli harici IP’lere açılmamalı veya en azından temel IP kısıtlaması veya ek kimlik doğrulaması uygulanmalıdır.

6. Test Süreci

turkce-sozluk-lab projesinde çeşitli test senaryoları uygulanabilir:

  • Birim Testleri: Sunucudaki fonksiyonları veya helper metotları ayrı ayrı test etmek.
  • Entegrasyon Testleri: /search, /search-like, /admin/add-word gibi endpoint’lerin gerçek bir MySQL test veritabanıyla birlikte düzgün çalışıp çalışmadığını kontrol etmek.
  • Yük/Performans Testleri: Yüksek sayıda eşzamanlı kelime araması veya admin işlemlerine sistemin tepkisini ölçmek (ör. JMeter veya Locust gibi araçlar).
  • Güvenlik Testleri: Kimlik doğrulama ve admin koruması (403 / 401 yanıtları), SQL enjeksiyona karşı hazırlıklı ifadeler, potansiyel XSS vb. kontrol edilir.

Nasıl çalıştırılır?

  • Eğer Mocha/Jest gibi bir test çerçevesi varsa, npm test benzeri komutla testler yürütülür.
  • Ayrı bir test veritabanı kullanarak, asıl verilerin bozulmasını engelleyebilirsiniz (DB_NAME=sozluk_test_db gibi).

Henüz proje içerisinde tüm testler dahil edilmemişse, test ekleyip eklemediğinize göre süreç şekillenir. Geliştirme tamamlandıkça CI/CD (GitHub Actions, GitLab CI, vb.) pipeline’ına ek test adımları koyarak otomasyon sağlanabilir.

7. Veritabanı Yapısı

Projenin temel tabloları:

  1. words
    • id (PK, int, auto increment)
    • word (VARCHAR)
    • origin (TEXT) – Kelimenin köken bilgisi
    • frequency_tags (VARCHAR) – S1, W1 gibi rozet bilgileri
    • audio_url (VARCHAR) – Ses dosyası bağlantısı (opsiyonel)
  2. definitions
    • id (PK, int, auto increment)
    • word_id (FK -> words.id)
    • definition_text (TEXT)
    • example_sentence (TEXT)
    • part_of_speech (VARCHAR) – Örn: noun, verb
    • sense_label (VARCHAR) – Örn: 1a, 1b gibi alt anlamları ayırt etme
  3. relations
    • id (PK, int, auto increment)
    • left_word_id (FK -> words.id)
    • right_word_id (FK -> words.id)
    • relation_type (VARCHAR) – Örn: synonym, antonym, idiom, quote, word_family, collocation

Ayrıca “full-text” arama için words.word ve/veya definitions.definition_text sütununa FULLTEXT indeksi eklenebilir. Bu sayede /search uç noktası MATCH...AGAINST sorgularını kullanarak daha isabetli sonuçlar verebilir.

9. Sıkça Sorulan Sorular (FAQ)

Soru: “Kelime Silme” butonu var fakat çalışmıyor. Neden?

Cevap: Kodda DELETE /admin/words/:id endpoint’inin henüz eklenmediği belirtilmiştir. Bu butonun backend tarafında bir karşılığı yok. İleriki sürümlerde implement edilecektir.

Soru: Full-text arama neden hiçbir sonuç döndürmüyor?

Cevap: MATCH ... AGAINST sorguları için MySQL’de bir FULLTEXT indeksi olması gerekir. Ayrıca varsayılan ayarlarda 4 karakterden kısa sözcükler göz ardı edilebilir. Bunu değiştirmek için innodb_ft_min_token_size ve ft_min_word_len gibi ayarları düzenleyip MySQL’i yeniden başlatabilirsiniz. Çok kısa kelimeler için proje, LIKE bazlı /search-like endpoint’ini kullanmayı önerir.

Soru: Admin girişinde kullanıcı adı mı şifre mi yanlış hatası alıyorum. .env dosyasında ADMIN_PASSWORD tanımlı ama giriş yapamıyorum.

Cevap: Kullanıcı adı sabit admin olarak kodlanmıştır. .env’deki ADMIN_PASSWORD değerini doğru yazdığınızdan emin olun. Örn: ADMIN_PASSWORD=Adm1nP@ssw0rd!. Ayrıca unutmamalısınız ki admin kullanıcı adı haricinde giriş kabul etmez (kodda öyle tasarlanmışsa). Dikkat: ADMIN_PASSWORD’de özel karakterler varsa, shell yorumlamasından dolayı .env formatına uygunluk kontrolü yapınız.

Soru: Aynı kelimeye birden çok definition ekleyebilir miyim?

Cevap: Evet. POST /admin/add-definition endpoint’inde word_name parametresi aynı olsa da her ekleme farklı “anlam” (satır) olarak kaydedilecektir. Örneğin “ruh” için “1a Akıl, akli denge” ve “1b birinin eylemlerini yöneten tutum” gibi birden çok definition olabilir.

Soru: Synonym eklemek istiyorum ama kelime yok hatası alıyorum.

Cevap: POST /admin/relations ile synonym (relation_type='synonym') eklenirken left_word_name ve right_word_name için words tablosunda kayıtlı kelimeler bulunamadığında hata döner. Önce kelimeleri ekleyip sonra aralarındaki ilişkiyi tanımlamalısınız.
