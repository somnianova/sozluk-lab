Genel Bakış

Proje Adı, Node.js ve MySQL kullanan bir Türkçe Sözlük uygulamasıdır. Kullanıcılar kelime arayabilir, detaylı tanımları ve örnek cümleleri görebilirler. Ayrıca, admin paneli üzerinden veritabanına yeni kelime ekleme, var olan kelimeleri düzenleme, anlam (definition) ekleme/silme/düzenleme gibi yönetimsel işlemler yapılabilir.

Öne çıkan özellikler:

- Kelimeler ve anlamlar üzerinde tam metin (full-text) veya LIKE tabanlı arama
- Admin paneli ile kelime, anlam (definition), ilişkiler (synonym, antonym vb.) yönetimi
- Oturum bazlı yetkilendirme ve basit güvenlik mekanizmaları (express-session)
- MySQL veritabanı ile ilişkisel tablolar (words, definitions, relations)
- Modüler ve geliştirici dostu yapı

Kurulum

Aşağıdaki adımları izleyerek projeyi kendi bilgisayarınızda veya bir sunucuda çalışır hale getirebilirsiniz:

1. Gereksinimler:
- Node.js (en az v14 veya üstü)
- MySQL 5.7+
- Git (opsiyonel, projeyi GitHub üzerinden klonlamak için)
2. Kaynak Kodun Edinilmesi:
- git clone <repo-url> komutu ile projeyi klonlayın (veya .zip olarak indirin).
- Proje klasörüne geçiş yapın: cd proje-adi
3. Bağımlılıkların Kurulması:
- npm install komutu ile gerekli paketleri (express, mysql2, express-session vb.) yükleyin.
4. MySQL Veritabanı Oluşturma:
- MySQL’de sozluk_db (veya .env dosyasında belirlediğiniz isimde) bir veritabanı oluşturun.
- words, definitions, relations tablolarını oluşturmak için SQL şemasını çalıştırın veya kendi taslağınızı kullanın.
5. Çevre Değişkenleri (.env) Düzenleme:
- Proje klasöründe .env dosyası (veya benzeri) bulunur. Burada DB_HOST, DB_USER, DB_PASSWORD, DB_NAME, ADMIN_PASSWORD, SESSION_SECRET, PORT gibi değerleri doldurun.
- Örnek .env içeriği:
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=mysql_sifreniz
DB_NAME=sozluk_db

ADMIN_PASSWORD=gucluSifre
SESSION_SECRET=oturumAnahtari
PORT=3000
6. Sunucuyu Başlatma:
- npm start veya node index.js komutu ile uygulamayı çalıştırın.
- Konsolda "Sunucu 3000 portunda çalışıyor..." benzeri bir mesaj görmelisiniz.
- Ardından, tarayıcınızda http://localhost:3000 adresine giderek uygulamayı test edebilirsiniz.

Veritabanı & Bağlantılar

Bu proje MySQL veritabanı kullanır. İlgili tablolar:

1. words
- id (INT, AUTO_INCREMENT, PK)
- word (VARCHAR) – Kelimenin kendisi
- origin (TEXT) – Köken bilgisi
- frequency_tags (VARCHAR) – Frekans rozetleri (S1, W1 vb.)
- audio_url (VARCHAR) – Ses dosyası bağlantısı
2. definitions
- id (INT, AUTO_INCREMENT, PK)
- word_id (INT, FK -> words.id) – Hangi kelimeye ait
- definition_text (TEXT) – Tanım
- example_sentence (TEXT) – Örnek cümle
- part_of_speech (VARCHAR) – Kelime türü (noun, verb vb.)
- sense_label (VARCHAR) – Anlam numaraları (örn: 1a, 2b)
3. relations
- id (INT, AUTO_INCREMENT, PK)
- left_word_id (INT, FK -> words.id)
- right_word_id (INT, FK -> words.id)
- relation_type (VARCHAR) – synonym, antonym, idiom vb.

Bağlantı kurulumu

index.js içerisinde şu şekilde bağlanır:

const db = mysql.createConnection({
host: process.env.DB_HOST,
user: process.env.DB_USER,
password: process.env.DB_PASSWORD,
database: process.env.DB_NAME
});

Sunucu başlarken MySQL’e bağlanır. Bağlantı kurulamazsa konsolda hata görürsünüz. Bağlantı bilgilerinin .env dosyasında doğru tanımlandığından emin olun.

API Kullanımı

Bu projede hem herkese açık (public) hem de admin (korumalı) uç noktalar (endpoint) mevcuttur. Aşağıda en yaygın kullanılanları listelenmiştir:

1. Public (Herkese Açık) API

• GET /search?kelime=...
Kelimeye göre arama yapar. Hem words.word hem de definitions.definition_text alanında arama yaparak eşleşen sonuçları döner.
Örnek İstek: GET /search?kelime=ruh
Örnek Yanıt (JSON veya HTML olarak dönebilir):
[
{
"word_id": 5,
"word": "ruh",
"definition_text": "Akıl, zihin, karakter anlamı..."
}
]
• GET /search-like?kelime=...
LIKE tabanlı arama yapar (daha basit ancak esneklik sağlar).
Örnek İstek: GET /search-like?kelime=ru
Dönen sonuçlar, "ru" içeren kelime veya tanımlara sahip girdiler olabilir.
• GET /api/synonyms?word=...
Belirli bir kelimenin eş anlamlılarını döner. relations tablosunda relation_type='synonym' olan ilişkiler üzerinden çalışır.
• GET /api/antonyms?word=...
Zıt anlamlı kelimeleri (antonym) döner.
• GET /api/idioms?word=...
Kelimenin ilişkili deyimlerini (idiom) döner.
• GET /api/quotes?word=...
Kelimenin ilişkili alıntılarını (quote) döner.
• GET /api/collocations?word=...
Kelimenin eşdizim (collocation) verilerini döner.
• GET /api/word-family?word=...
Kelimenin ait olduğu kelime ailesi (word_family) ilişkilerini döner.
• GET /words-by-letter?letter=...
Belirli bir harfle başlayan kelimeleri listeler.
(Döndürdüğü veriler JSON formatında olabilir. Frontend bunu işleyip kullanıcıya gösterir.)

2. Admin (Koruma Altında) API

Bu uç noktalar, /admin/login üzerinden giriş yapmış admin oturumunu gerektirir. checkAdmin adlı middleware ile korunurlar. Eğer oturum açılmamışsa 403 hata döner.

• GET /admin/login
Basit bir login formu sunar (HTML). Kullanıcı adı admin, şifre .env içindeki ADMIN_PASSWORD değeri olmalıdır.
• POST /admin/login
Formdan gelen kullanıcı adı ve şifreyi kontrol eder. Doğruysa req.session.isAdmin = true ayarlayarak admin oturumu açar.
• GET /admin/logout
Admin oturumunu sonlandırır (session’ı yok eder).
• POST /admin/add-word
Yeni kelime ekler (body’de word, origin, frequency_tags, audio_url vb. bekler).
Örnek Body (JSON):
{
"word": "ruhsal",
"origin": "Arapça 'rūḥ'",
"frequency_tags": "S1,W1",
"audio_url": "/audio/ruhsal.mp3"
}
• GET /admin/words
Kelime listeleme + sayfalama. ?page=1&limit=5&sort=word_asc parametrelerini destekler. JSON döner.
• GET /admin/words/:id
Belirli bir kelimenin verilerini ve ilk definition’ını döner.
• PUT /admin/words/:id
Kelimeyi günceller (word, origin, frequency_tags, audio_url gibi alanlar).
• DELETE /admin/words/:id (Şu an taslak durumda, uygulamada henüz tam entegre değil olabilir.)
• GET /admin/definitions
Definitions listeleme + sayfalama + sıralama (definition_id_desc, word_asc vb.). JSON olarak döner.
• POST /admin/add-definition
Var olan bir kelimeye yeni anlam ekler. Body’de word_name, definition_text, example_sentence, part_of_speech, sense_label gibi alanlar beklenir. word_name üzerinden words tablosu ile eşleştirme yapılır.
• PUT /admin/definitions/:id
Bir definition kaydını günceller (örnek cümle, tanım metni vb.).
• DELETE /admin/definitions/:id
Bir definition kaydını siler.
• GET /admin/relations
relations tablosundaki tüm ilişkileri listeler.
• POST /admin/relations
İki kelime arası ilişki ekler. Body’de left_word_name, right_word_name, relation_type (synonym, antonym vb.) bekler.
• DELETE /admin/relations/:id
İlgili ilişki kaydını siler.
• GET /admin/last-words
Son eklenen 10 kelimeyi (veya istenen sayıda) döner; admin paneli "Son Eklenenler" bölümünde kullanılır.

Bu uç noktalar, admin panelindeki formlarla entegre çalışacak şekilde tasarlanmıştır.

Admin Paneli Kullanımı

1. Admin Giriş:
- Uygulama çalışırken http://localhost:3000/admin/login adresine gidin.
- Varsayılan kullanıcı: admin, şifre: .env içindeki ADMIN_PASSWORD.
- Giriş başarılı olursa tarayıcınıza bir session cookie eklenir.
2. Ana Panel (Dashboard):
- admin.html üzerinden erişilir. Oturum açtıktan sonra admin arayüzü görünür.
- Son eklenen kelimeler, kelime ekleme formu, defitions listesi, ilişkiler gibi sekmeler veya menü öğeleri bulunur.
3. Kelime Ekleme (Words Section):
- Admin panelinde "Kelime Ekle" sekmesine gidin.
- word, origin, frequency_tags, audio_url gibi alanları doldurun.
- Gönderdiğinizde, backend /admin/add-word endpoint’ini çağırır ve kaydı veritabanına ekler.
4. Definition Ekleme (Definitions Section):
- Admin panelinde "Definitions" sekmesine gidin.
- Burada yeni bir definition eklemek için form mevcuttur. Kelime Adı (word_name), definition_text, example_sentence gibi alanları doldurarak gönderin.
- /admin/add-definition ile veritabanına eklenir. Kelime adı, words tablosundaki bir kayda karşılık gelmelidir.
5. Kayıt Düzenleme / Silme:
- Her liste öğesinin yanında "Düzenle" (edit) veya "Sil" (delete) butonları olabilir.
- Düzenle butonuna tıkladığınızda, modal pencere açılarak form verilerini güncelleyebilirsiniz. POST/PUT istekleri ile backend’e iletilir.
- Silme butonuna tıkladığınızda, ilgili ID’ye göre DELETE isteği atanır (örn. /admin/definitions/:id).
6. İlişkiler Yönetimi (Relations Section):
- synonym, antonym, idiom, quote, collocation gibi ilişki türleri eklenebilir.
- Kelimenin adını yazarak soldaki ve sağdaki kelimeleri belirleyin, ilişki türünü seçin, ardından ekleyin.
- Listeyi görebilir, sil butonu ile istemediğiniz ilişkileri kaldırabilirsiniz.
7. Çıkış (Logout):
- http://localhost:3000/admin/logout adresine giderek session’ı sonlandırabilirsiniz.
- Bu işlemin ardından admin yetkisi kalkar ve korumalı endpoint’lere yeniden girmek için tekrar giriş yapmanız gerekir.

Geliştirme Rehberi

Projeye katkıda bulunmak veya yeni özellikler eklemek isteyenler için:

1. Geliştirme Ortamı Kurun:
- Kendi MySQL veritabanınızı oluşturun, .env dosyanızda o veritabanına göre ayarlar yapın.
- Üretim (production) veritabanına doğrudan yazma riskini önlemek için daima yerel/kendi test veritabanınızı kullanın.
2. Kod Düzeni & Standartlar:
- Mevcut örnek kodu (örn. index.js) inceleyin. Gelişigüzel değişikliklerden kaçının; proje yapısına uyumlu geliştirin.
- Gerekirse controllers/ veya routes/ gibi klasörler oluşturabilir, mantığınızı buraya bölebilirsiniz. Ancak basit projelerde her şey tek dosyada da olabilir.
3. Yeni Özellik Geliştirme:
- Yeni bir dal (branch) oluşturun: git checkout -b feature/...
- Kodlamanızı tamamlayıp test ettikten sonra, Pull Request (PR) açıp takım arkadaşlarınızdan veya proje sahibinden inceleme isteyin.
4. Hata Ayıklama (Debugging):
- console.log() veya Node.js debugger (node --inspect) kullanarak kodu adım adım inceleyin.
- MySQL sorgularında hata alırsanız, parametrelerin doğru olup olmadığını ve veritabanı tablolarının mevcut durumda olduğunu kontrol edin.
5. Test & Onay:
- Değişiklikleri yerel ortamda test edin. Kelime ekleme, arama yapma, admin panel üzerinden denemeler gibi senaryoları test ederek hataları azaltın.
- Eğer proje test framework’leri (ör. Mocha, Jest) içeriyorsa, testleri çalıştırın ve yeni eklediğiniz kod için ek testler yazın.
6. Dokümantasyon Güncellemesi:
- Yeni endpoint veya alan eklediğinizde, bu rehberi (veya README.md vb.) güncelleyerek belgelendirin.
- Böylece sonraki katkıda bulunanlar da projeyi daha rahat anlayabilir.

Güvenlik Önlemleri

1. Admin Girişi & Session Yönetimi:
- express-session kullanılır. Oturum açıldığında req.session.isAdmin = true şeklinde bir bayrak set edilir.
- Tüm /admin/ rotaları checkAdmin middleware’i ile korunur. Bu da req.session.isAdmin yoksa 403 döndürür.
- Üretim ortamında HTTPS (SSL) kullanarak session cookie’lerin güvenliğini artırmak önerilir.
2. Parola Koruması:
- Admin şifresi .env içerisinde tanımlıdır. Varsayılan şifreleri veya örnek şifreleri asla canlı sistemde kullanmayın. Güçlü bir parola seçin.
- İleride çok kullanıcı veya rollü bir yapı istenirse, parolaları veritabanında şifreli (bcrypt vb.) şekilde saklayacak gelişmiş kullanıcı/role sistemi eklenebilir.
3. Veri Doğrulama (Validation):
- Kullanıcı girişlerinin (kelime, tanım metni vb.) doğrulanması, olası SQL Injection vb. risklerini azaltır. Projede parametreli sorgular (?) kullanılmaktadır, bu iyi bir uygulamadır.
- Gerekiyorsa ek validasyon kütüphaneleri (ör. express-validator) ekleyerek form girişlerini daha katı şekilde denetleyebilirsiniz.
4. XSS & CSRF Önlemleri:
- Admin panelinde girilen veriler geri yansıtılırken HTML içeriği escape edilmeli veya gerekli sanitizasyon yapılmalıdır. Aksi halde potansiyel XSS zafiyeti oluşabilir.
- CSRF için, örneğin csurf gibi bir middleware kullanmak düşünülebilir. Şu an proje basit bir admin yapısına sahip olduğundan minimal seviye güvenlik var.
5. HTTPS (Üretimde Önerilir):
- Session çerezleri (cookie) HTTP üzerinden aktarılırsa riskli olabilir. SSL sertifikası ile (ör. Let’s Encrypt) HTTPS kurulumu yapmak, güvenliği artırır.

Sürüm Kontrolü & Dağıtım

1. Git Workflow:
- Yeni özellik veya hata düzeltmeleri için ayrı dallar (branch) kullanın (feature/..., bugfix/... gibi).
- Tamamlayınca Pull Request açarak gözden geçirme (review) ve birleştirme (merge) yapın.
- Her commit mesajında özlü ama açıklayıcı ifadeler kullanın.
2. Dağıtım Seçenekleri:
- Manuel Sunucu: Bir Linux sunucu (VPS, dedi) üzerinde Node.js kurarak bu projeyi git pull ile çekebilir, .env ayarlarını yapabilir, npm install ve npm start komutlarıyla çalıştırabilirsiniz. Sürekli çalışması için pm2 veya screen kullanılabilir.
- Platform as a Service (PaaS): Heroku, Railway, Render gibi platformlarda otomatik dağıtım yapılabilir. .env değişkenlerini bu platformların “Config Vars” veya “Environment” ayarlarından eklemeniz gerekir.
- Docker: Bir Dockerfile mevcutsa docker build ve docker run ile container oluşturabilirsiniz. docker-compose.yml varsa veritabanı ve uygulamayı birlikte çalıştırmak kolay olur.
- Bulut Sağlayıcılar: AWS, Azure veya GCP üzerinde “Node.js App” olarak dağıtım yapılabilir. Veritabanı olarak RDS (AWS) veya Cloud SQL (GCP) gibi yönetilen MySQL hizmetleri kullanılabilir.
3. Güncellemeler ve Yedekler:
- Yeni bir sürüm çıkmadan önce veritabanınızı mutlaka yedekleyin. Tablolarda değişiklik yapmak gerekiyorsa (ör. yeni sütun ekleme) migration script’leri hazırlayabilirsiniz.
- Her sürümde, CHANGELOG dosyasını gözden geçirerek eklenen/çıkarılan özellikleri takip edin.
4. Üretim Ortamında Dikkat Edilmesi Gerekenler:
- NODE_ENV değişkenini production olarak ayarlamak.
- Yanıtları cache’leme, sıkıştırma (compression) vb. performans optimizasyonları eklenebilir.
- Uygulamayı mutlaka bir reverse proxy (Nginx, Caddy vb.) üzerinden HTTPS ile yayınlamak.
- Session store olarak redis gibi bir dış servis kullanmak (server yeniden başlatıldığında session kaybını önlemek için).
